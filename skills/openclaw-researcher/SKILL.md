# OpenClaw Researcher Skill

## 功能定位

在 openclaw-writer 動手寫文章之前，先弄清楚兩件事：

1. 目標受眾在這個主題上，實際在搜尋什麼、在問什麼
2. 他們的真實需求是什麼，而不是我們猜測的需求

輸出一份結構化的受眾需求報告，交給 writer 用。

---

## Pipeline 位置

```
PM（任務指派）
  → researcher（你，產出受眾需求報告）← 你在這裡
  → writer（用報告寫文章）
  → publisher（發布）
```

---

## 輸入

- 主題關鍵字（必要）
- 受眾層級（選用：初階 / 中階 / 高階）

---

## 執行流程（五步）

**Step 1：搜尋受眾在問什麼**

用以下幾個角度搜尋，每個角度搜一次，記錄找到的真實問題：

- `[主題] how to`（做法類搜尋）
- `[主題] vs`（比較類搜尋）
- `[主題] problem` 或 `[主題] 問題`（痛點類搜尋）
- `[主題] for beginners` 或 `[主題] 初學者`（入門類搜尋）
- `[主題] project` 或 `[主題] 應用`（應用類搜尋）

搜尋工具：WebSearch

**Step 2：分析搜尋意圖**

把找到的問題分成三類：
- **想做某件事**（要程式碼、要接線圖、要步驟）
- **想理解某件事**（要原理解釋、要概念釐清）
- **想做選擇**（要比較、要選型建議）

判斷這個主題的受眾主要屬於哪一類，或各類的比例。

**Step 3：找出 SEO 關鍵字**

從搜尋結果中整理：
- 主要關鍵字（出現最多次的詞）
- 長尾關鍵字（具體問題、具體情境）
- 相關詞（受眾搜尋這個主題時，同時也在搜尋的東西）

**Step 4：定位受眾需求**

根據搜尋意圖，判斷：
- 受眾現在卡在哪裡？
- 他們做完這件事之後想達到什麼目標？
- 他們最常遇到的具體問題是什麼？

**Step 5：輸出報告**

產出結構化受眾需求報告（見「輸出格式」）

---

## 輸出格式

```markdown
# 受眾需求報告：[主題]

## 主要搜尋意圖
[想做 / 想理解 / 想選擇，各佔比例的判斷]

## 受眾最常問的問題（前5-8個）
1. [問題一]
2. [問題二]
...

## SEO 關鍵字

**主要關鍵字：**
- [關鍵字1]
- [關鍵字2]

**長尾關鍵字：**
- [長尾詞1]
- [長尾詞2]

**相關詞：**
- [相關詞1]
- [相關詞2]

## 受眾當前痛點
[2-3句，說明受眾卡在哪裡、為什麼卡]

## 受眾目標
[1-2句，說明他們做完這件事想達到什麼]

## 建議文章角度
[根據以上分析，建議 writer 從什麼角度切入，能最直接命中需求]

## 建議標題選項（3個）
1. [標題一]
2. [標題二]
3. [標題三]
```

## 搜尋策略補充

搜尋時特別關注這些來源：

- **Reddit 相關 subreddit**（arduino, raspberry_pi, DIY, electronics 等）
  實際的使用者問題，比官方文件更能反映真實需求

- **YouTube 影片標題與留言**
  高觀看數的影片標題揭示什麼樣的問題有人在找

- **Stack Overflow / Arduino Forum**
  具體的技術問題，反映中階受眾的卡點

- **Google Autocomplete 邏輯**
  搜尋框的自動補全反映最常見的搜尋行為

不需要每個來源都去，但要確保至少覆蓋「使用者真實問題」和「技術問答」這兩個角度。
