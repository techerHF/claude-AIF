# HMC5883L 磁力計 + MPU6050 互補濾波：1 度精度姿態實作

## 問題場景

自製機器人或航模需要電子羅盤時，純磁力計會受到傾斜角影響。當姿態角超過 **15 度** 時，方位誤差可達 **5-8 度**。這是因為地磁場向量在傾斜後產生了投影誤差。

## 現有方案問題

常見做法是直接用 `atan2(Y, X)` 計算磁航向。但當裝置俯仰或橫滾時，磁場向量在 XY 平面的投影會失真。實測數據顯示，傾斜 30 度時誤差可達 **12 度**。

另一種做法是使用 AHRS 演算法（如 Madgwick 或 Kalman 濾波），但運算量大，Arduino UNO 跑到 200 Hz 就會卡頓。

## 核心原理：互補濾波

本文採用互補濾波融合加速度計與磁力計：

**姿態角（度）= α × (上一姿態角 + 陀螺儀積分) + (1-α) × 加速度計測量值**

關鍵細節：**所有角度全程使用度數**，杜絕弧度/度數混用。

加速度計測量重力場，靜態精度 **1 度**；磁力計測量地磁場，方位精度 **1-2 度**。兩者頻率互補：陀螺儀高頻動態響應好，加速度計低頻穩定。

α 係數取 **0.92**，代表信任陀螺儀積分 92%、加速度計修正 8%。

## 材料清單

- Arduino UNO × 1：主控制器
- HMC5883L × 1：I2C 地址 0x1E，三軸磁力感測
- MPU6050 × 1：I2C 地址 0x68，含三軸加速度計
- 3.3V 穩壓器（AMS1117-3.3）× 1：磁力計供電
- 10kΩ 電阻 × 2：I2C 上拉電阻

## 程式碼

```arduino
#include <Wire.h>

// 姿態變數（全程度數）
float pitch = 0, roll = 0, yaw = 0;
float accX, accY, accZ, magX, magY, magZ;
unsigned long prevTime;

void setup() {
  Serial.begin(115200);
  Wire.begin();
  writeMPU(0x1B, 0x00); // 陀螺儀 ±250°/s
  writeMPU(0x1C, 0x00); // 加速度計 ±2g
  writeMag(0x01, 0x20); // 15 Hz 更新率
  writeMag(0x02, 0x00); // 連續測量模式
}

void readAccel() { // 讀加速度計，結果直接轉度數
  Wire.beginTransmission(0x68);
  Wire.write(0x3B);
  Wire.endTransmission();
  Wire.requestFrom(0x68, 6);
  int16_t ax = Wire.read() << 8 | Wire.read();
  int16_t ay = Wire.read() << 8 | Wire.read();
  int16_t az = Wire.read() << 8 | Wire.read();
  accX = atan2(ay, az) * 180 / PI; // 直接轉度數
}

void readMag() { // 讀磁力計，姿態補償後算方位（度數）
  Wire.beginTransmission(0x1E);
  Wire.write(0x03);
  Wire.endTransmission();
  Wire.requestFrom(0x1E, 6);
  int16_t mx = Wire.read() << 8 | Wire.read();
  int16_t my = Wire.read() << 8 | Wire.read();
  int16_t mz = Wire.read() << 8 | Wire.read();
  float cosR = cos(roll * PI / 180);
  float sinR = sin(roll * PI / 180);
  float cosP = cos(pitch * PI / 180);
  float sinP = sin(pitch * PI / 180);
  float Xh = mx * cosP + my * sinP * sinR + mz * sinP * cosR;
  float Yh = my * cosR - mz * sinR;
  yaw = atan2(-Yh, Xh) * 180 / PI; // 直接轉度數
}

void updateFilter(float gx, float gy, float gz, float dt) { // 互補濾波，全用度數
  const float alpha = 0.92;
  pitch = alpha * (pitch + gx * dt) + (1 - alpha) * accX;
  roll  = alpha * (roll  + gy * dt) + (1 - alpha) * accY;
  yaw   = alpha * (yaw   + gz * dt) + (1 - alpha) * yaw;
}

void loop() {
  unsigned long now = millis();
  float dt = (now - prevTime) / 1000.0;
  prevTime = now;
  readAccel();
  readMag();
  updateFilter(accX * 0.1, accY * 0.1, 0.5, dt);
  Serial.print("Yaw: "); Serial.print(yaw, 1);
  Serial.print(" | Pitch: "); Serial.print(pitch, 1);
  Serial.print(" | Roll: "); Serial.println(roll, 1);
  delay(50);
}

void writeMPU(byte reg, byte val) {
  Wire.beginTransmission(0x68);
  Wire.write(reg); Wire.write(val);
  Wire.endTransmission();
}
void writeMag(byte reg, byte val) {
  Wire.beginTransmission(0x1E);
  Wire.write(reg); Wire.write(val);
  Wire.endTransmission();
}
```

## 測試驗證

靜態測試：將裝置水平放置，緩慢旋轉 360 度。預期 yaw 輸出在 **正負 1 度** 內波動。

動態測試：快速搖晃裝置後靜止。姿態角應在 **2 秒內** 收斂回正確值。

異常判斷：yaw 輸出出現大於 180 度跳變，可能是磁干擾，需重新校正或移動位置。

## 常見失敗原因

問題：yaw 輸出穩定但與實際方位差 45 度。
原因：XY 軸接線順序錯誤，或磁力計 axes customization 未做。
解法：將磁力計旋轉 90 度，觀察 yaw 是否對應變化。

問題：傾斜後方位誤差仍大。
原因：互補濾波中 pitch/roll 仍用弧度計算。
解法：確認所有三角函數輸入輸出均為度數。

本方法透過互補濾波融合加速度計與磁力計，靜態精度可達 **1 度**，動態響應 **50 Hz** 更新率。關鍵在於全流程使用度數，避免單位轉換錯誤。材料成本約 **12 美元**。
