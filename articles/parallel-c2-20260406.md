# HMC5883L 磁力計電子羅盤：姿態補償與互補濾波實作

## 問題場景

機器人在斜坡上行駛時，電子羅盤指向偏離了 23 度。操作者以為目標在左側，實際卻需向右修正。這不是羅盤壞了，而是姿態改變了地磁場測量軸向。

三軸磁力計輸出會隨著 X、Y、Z 各軸相對於地磁場的夾角改變。無姿態補償時，側傾 30 度的機器人測得的地磁場水平分量減少約 13%。以此計算出的方位角誤差可達 15 度以上。

## 現有方案的問題

多數 HMC5883L 教學僅做以下處理：讀取原始 X、Y 值，套用 atan2(Y, X) 得到方位角。這在感測器保持水平時尚可接受。

問題在於：姿態變化時，各軸量測到的地磁場分量會重新投影。這種投影偏差不是校正能消除的。此外，純磁力計數值本身含有隨機飄移，約 2-5 度 per minute，室內環境還有軟磁物質干擾。

基礎教學通常建議「遠離金屬」或「定期重新校正」，但缺乏量化依據。使用者在複雜環境中仍無法判断誤差不會影響應用。

## 核心原理：姿態補償與互補濾波

三軸磁力計測得的地磁場向量為 B = [Bx, By, Bz]。當感測器有側傾角 roll 和俯仰角 pitch 時，水平面的地磁場分量需要姿態矩陣旋轉：

```
B_horizontal = R(pitch) * R(roll) * B
```

其中 R(pitch) 為俯仰旋轉矩陣，R(roll) 為側傾旋轉矩陣。旋轉後取 X 和 Y 分量，再 atan2 得出方位角。

然而，加速度計在低速運動時能可靠估測姿態，但高頻雜訊較大。磁力計在姿態變化時投影偏差顯著，但低頻穩定性佳。互補濾波將兩者融合：

```
roll_filtered = α * roll_accel + (1-α) * roll_mag
pitch_filtered = α * pitch_accel + (1-α) * pitch_mag
```

實務上 α 取 0.8-0.95，取決於運動頻率。本專案使用 α = 0.92，高頻姿態由加速度計主導，低頻飄移由磁力計修正。

## 材料清單

- Arduino UNO × 1：主控制器
- HMC5883L 三軸磁力計模組 × 1：I2C 位址 0x1E，量程 ±8 Gauss
- MPU-6050 三軸加速度計 × 1：I2C 位址 0x68，用於姿態估算
- 麵包板 × 1：原型搭建
- 杜邦線 × 若干：接线

HMC5883L 模組建議選用 GY-271 或 QMC5883L 相容版本，兩者接腳相容但晶片不同，驅動函式庫略有差異。

## 製作步驟

步驟1：接线
HMC5883L SDA → A4，SCL → A5，VCC → 3.3V，GND → GND
MPU-6050 SDA → A4，SCL → A5，VCC → 5V，GND → GND
兩感測器共用 I2C 匯流排。

步驟2：安裝函式庫
Arduino IDE → 草稿碼 → 匯入函式庫 → 管理函式庫
搜尋「Adafruit HMC5883 Unified」與「I2Cdev」並安裝。

步驟3：姿態矩陣計算
利用加速度計估算初始 roll 和 pitch：
```
roll_accel = atan2(ay, az) * RAD_TO_DEG
pitch_accel = atan2(-ax, sqrt(ay² + az²)) * RAD_TO_DEG
```

步驟4：應用互補濾波
將磁力計計算的姿態角與加速度計估算角融合，α = 0.92。

步驟5：投影修正
使用旋轉矩陣將地磁場向量投影至水平面，再計算 atan2 方位角。

## 程式碼

```arduino
// 檔案說明：HMC5883L 姿態補償電子羅盤，結合 MPU-6050 加速度計互補濾波
// 作者：HF Chang
// 硬體：Arduino UNO + HMC5883L + MPU-6050，共用 I2C 匯流排
// 接線：HMC5883L SDA→A4, SCL→A5; MPU-6050 SDA→A4, SCL→A5

#include <Wire.h>
#include <math.h>

// HMC5883L 暫存器
#define HMC5883L_ADDRESS 0x1E
#define HMC5883L_REG_CONFIG_A 0x00
#define HMC5883L_REG_MODE 0x02
#define HMC5883L_REG_DATA 0x03

// MPU-6050 暫存器
#define MPU6050_ADDRESS 0x68
#define MPU6050_REG_ACCEL_X 0x3B

// 互補濾波係數：高頻由加速度計主導
const float ALPHA = 0.92;

// 濾波後姿態角
float roll_filtered = 0;
float pitch_filtered = 0;

void setup() {
  Serial.begin(9600);
  Wire.begin();

  // 初始化 HMC5883L：一般測量模式，平均值 4，I2C 100kHz
  Wire.beginTransmission(HMC5883L_ADDRESS);
  Wire.write(HMC5883L_REG_CONFIG_A);
  Wire.write(0x70);  // bit[7:6]=11 平均4次, bit[5:4]=11 75Hz, bit[1:0]=00 正常測量
  Wire.endTransmission();

  Wire.beginTransmission(HMC5883L_ADDRESS);
  Wire.write(HMC5883L_REG_MODE);
  Wire.write(0x00);  // 連續測量模式
  Wire.endTransmission();

  // 初始化 MPU-6050
  Wire.beginTransmission(MPU6050_ADDRESS);
  Wire.write(0x6B);  // PWR_MGMT_1
  Wire.write(0x00);  // 喚醒，時脈設定為 X 軸 Gyro 參考
  Wire.endTransmission();

  Serial.println("Heading (deg),Roll (deg),Pitch (deg)");
}

void loop() {
  // === 讀取加速度計 ===
  int16_t ax, ay, az;
  readMPU6050Accel(ax, ay, az);

  // 加速度計估算姿態角（基準角）
  float roll_accel = atan2((float)ay, (float)az) * 180.0 / PI;
  float pitch_accel = atan2((float)-ax, sqrt((float)ay * ay + (float)az * az)) * 180.0 / PI;

  // === 讀取磁力計 ===
  int16_t mx, my, mz;
  readHMC5883L(mx, my, mz);

  // 磁力計估算姿態角（用於互補濾波）
  float roll_mag = atan2((float)mz, (float)my);
  float pitch_mag = atan2((float)-mx, sqrt((float)my * my + (float)mz * mz));

  // === 互補濾波 ===
  roll_filtered = ALPHA * roll_accel + (1.0 - ALPHA) * roll_mag * 180.0 / PI;
  pitch_filtered = ALPHA * pitch_accel + (1.0 - ALPHA) * pitch_mag * 180.0 / PI;

  // === 姿態補償：將磁力計資料投影至水平面 ===
  // 俯仰角 rad
  float pitch_rad = pitch_filtered * PI / 180.0;
  float roll_rad = roll_filtered * PI / 180.0;

  // 投影後的水平分量
  float Bx_h = mx * cos(pitch_rad) + my * sin(roll_rad) * sin(pitch_rad) - mz * cos(roll_rad) * sin(pitch_rad);
  float By_h = my * cos(roll_rad) + mz * sin(roll_rad);

  // 方位角
  float heading = atan2(By_h, Bx_h) * 180.0 / PI;
  if (heading < 0) heading += 360.0;

  Serial.print(heading, 1);
  Serial.print(",");
  Serial.print(roll_filtered, 1);
  Serial.print(",");
  Serial.println(pitch_filtered, 1);

  delay(100);
}

void readMPU6050Accel(int16_t& ax, int16_t& ay, int16_t& az) {
  Wire.beginTransmission(MPU6050_ADDRESS);
  Wire.write(MPU6050_REG_ACCEL_X);
  Wire.endTransmission(false);
  Wire.requestFrom(MPU6050_ADDRESS, 6);

  if (Wire.available() >= 6) {
    ax = (Wire.read() << 8) | Wire.read();
    ay = (Wire.read() << 8) | Wire.read();
    az = (Wire.read() << 8) | Wire.read();
  }
}

void readHMC5883L(int16_t& mx, int16_t& my, int16_t& mz) {
  Wire.beginTransmission(HMC5883L_ADDRESS);
  Wire.write(HMC5883L_REG_DATA);
  Wire.endTransmission(false);
  Wire.requestFrom(HMC5883L_ADDRESS, 6);

  if (Wire.available() >= 6) {
    // HMC5883L 資料格式：MSB 在前，X、Z、Y 順序
    mx = (Wire.read() << 8) | Wire.read();
    mz = (Wire.read() << 8) | Wire.read();
    my = (Wire.read() << 8) | Wire.read();
  }
}
```

上傳方式：Arduino IDE 1.8+ 將程式碼貼入，上傳至 UNO。Serial Monitor 設定 9600 baud。

預期輸出範例：
```
Heading (deg),Roll (deg),Pitch (deg)
182.3,2.1,-1.4
183.1,2.3,-1.2
```

每列依序為方位角、側傾角、俯仰角。方位角 0-360 度，側傾和俯仰角 ±90 度。

## 測試驗證

將感測器水平放置，緩慢旋轉 360 度。正常情形下方位角應連續變化，不應有突波。室內環境中，封閉房間與金屬門框會造成 5-15 度區域性偏移。

姿態測試：固定方位角為 90 度（東），將模組側傾 30 度。經補償後，方位角應維持在 82-98 度範圍內。未補償版本可能顯示 65-75 度。

漂移測試：靜置 5 分鐘後，方位角飄移應小於 3 度。若超過，代表軟磁干擾或初始化尚未穩定，需重新校正。

## 常見失敗原因

問題：方位角固定不變或完全錯誤
原因：I2C 接線鬆脫，或 SDA/SCL 順序錯誤
解法：用 I2C Scanner 確認位址，HMC5883L 應回應 0x1E，MPU-6050 為 0x68

問題：側傾角 close to 90 度或 NaN
原因：加速度計測量值為零（姿態恰好垂直於某軸）
解法：避免將感測器 Z 軸完全垂直放置，或在程式中限制 pitch 角度範圍

問題：方位角每分鐘漂移超過 5 度
原因：磁力計周圍有軟磁物質（喇叭、馬達）或硬磁干擾（永久磁鐵）
解法：重新選擇安裝位置，遠離含鐵金屬，或使用橢球校正演算法修正 offset

## 延伸方向

本案可結合 GPS 模組，在戶外實現無累積誤差的電子羅盤。GPS 提供移動方向，磁力計提供絕對方位，兩者互補可應用於除草機或自動搬運車等戶外自動導航系統。

另一方向為加入磁力計工廠校正參數自動擷取，結合本篇文章的互補濾波框架，製作成可攜式電子羅盤工具，適用於室內緊急定位或地質考察。
