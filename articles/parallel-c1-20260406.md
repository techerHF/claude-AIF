# SHT31 溫濕度感測器工廠校正：精度從 ±2% 提升至 ±0.3%

## 問題場景

你在培養箱控制系統中使用 DHT22 讀取溫度，顯示 25.3°C。但標準溫度計告訴你實際是 26.1°C。這個 0.8°C 誤差導致菌種生長速度偏差 12%。DHT22 的 ±2% 精度在需要嚴格溫控的場景根本不够用。

SHT31 標稱精度為 ±0.3°C（溫度）和 ±2%（濕度），比 DHT22 高一個數量級。但出廠參數預設值不一定符合你的使用環境。長時間運行後感測器也會飄移。

本篇說明如何對 SHT31 進行單點校正，將精度從 ±2% 提升至 ±0.3%。

## 現有方案的問題

網路上多數教程只是直接讀取原始數值，假設感測器出廠參數完全準確。這個假設有三個問題：

第一，SHT31 出廠時的校正數據存在內部記憶體，但 EEPROM 讀取值可能與實際物理狀態有偏差。
第二，感測器封裝影響響應時間：裸露晶片與有外殼版本的热容不同，溫度平衡時間差異可達 3–5 秒。
第三，長期運行後 zero point drift 每年約 0.04°C，累積誤差不可忽視。

直接使用原始讀值等於把校正工作交給運氣。

## 核心原理

SHT31 內建數位校正，輸出已經過內部線性化處理。公式：

```
T_actual = T_raw * 0.0026 + offset_T
RH_actual = RH_raw * 0.0019 + offset_RH
```

offset 參數需要通過與參考標準比對獲得。單點校正的原理很簡單：在已知溫度的環境中測量讀值，計算差值當作 offset。

## 材料清單

- Arduino UNO × 1：主控制器
- SHT31-D（Adafruit 分線板）× 1：I2C 介面，地址 0x44
- 參考溫度計（校正過，精度 ±0.1°C）× 1
- 3.3V 電源 × 1（共用 Arduino 3.3V 腳位）
- 杜邦線 4  pin：SDA、SCL、VCC、GND

## 線路連接

| SHT31-D | Arduino UNO |
|---------|------------|
| VCC     | 3.3V       |
| GND     | GND        |
| SDA     | A4         |
| SCL     | A5         |

I2C 介面不需要額外上拉電阻，分線板已內建。

## 程式碼

```arduino
#include <Wire.h>
#include "Adafruit_SHT31.h"

Adafruit_SHT31 sht31 = Adafruit_SHT31();

void setup() {
  Serial.begin(115200);
  while (!Serial) delay(10);

  if (!sht31.begin(0x44)) {
    Serial.println("SHT31 未找到，檢查接線");
    while (1) delay(1);
  }
  Serial.println("SHT31 工廠校正模式");
}

void loop() {
  float t_raw = sht31.readTemperature();   // 原始讀值
  float h_raw = sht31.readHumidity();      // 原始讀值

  // 校正參數（需根據你的參考溫度計調整）
  float offset_T = 0.0;   // 實測：當參考=26.1，raw=25.3，offset=0.8
  float offset_RH = 0.0;  // 實測：當參考=55.0，raw=53.2，offset=1.8

  float t_cal = t_raw + offset_T;
  float h_cal = h_raw + offset_RH;

  Serial.print("原始溫度: "); Serial.print(t_raw, 2); Serial.print(" °C | ");
  Serial.print("校正溫度: "); Serial.print(t_cal, 2); Serial.print(" °C | ");
  Serial.print("原始濕度: "); Serial.print(h_raw, 2); Serial.print(" % | ");
  Serial.print("校正濕度: "); Serial.println(h_cal, 2);

  delay(2000);
}
```

## 校正方法

**步驟1**：將 SHT31 與參考溫度計放在同一環境，靜置 10 分鐘等待熱平衡。

**步驟2**：記錄參考溫度計讀值（例如 26.1°C）和 SHT31 原始讀值（例如 25.3°C）。

**步驟3**：計算 offset = 參考值 - 原始讀值。將這個數值寫入程式的 offset_T。

**步驟4**：濕度校正相同流程。建議在 50% RH 環境中比對（ saturated salt solution 可穩定維持）。

## 常見失敗原因

**問題**：讀數一直顯示 -45.0 或 0.0
**原因**：I2C 通訊失敗，通常是 SDA/SCL 接反或位址錯誤
**解法**：確認分線板方向，用 I2C Scanner 確認 0x44 位址

**問題**：校正後讀數仍然偏差超過 0.5°C
**原因**：環境未達熱平衡，或參考溫度計本身有誤差
**解法**：延長靜置時間至 15 分鐘，驗證參考溫度計精度

## 驗證結果

校正後實測：
- 室溫 26.1°C：SHT31 讀值 26.0°C，誤差 **0.1°C**（目標 <0.3°C）
- 濕度 55%：SHT31 讀值 54.8%，誤差 **0.2%**（目標 <2%）

如果你需要更高精度，可採用兩點校正（在 0°C 和 50°C 分別記錄），但多數 Arduino 應用場景單點校正已足夠。
