## 走廊燈在你伸手之前就亮了

半夜走廊，你還沒有摸到牆壁，燈就亮了。

不是聲控，不是定時器，也不是有人幫你按的。就是自動亮了，因為你走進去了。

這背後用的東西叫做 PIR 感測器。它在你家裡可能已經裝了好幾個。走廊燈、保全警報器、自動沖水的廁所，很多都是它在做事。但是大多數人沒有想過，它到底是怎麼「感覺」到你的。

這篇文章從原理說起，然後用 HC-SR501 模組和 Arduino 做出一個可以跑的 prototype。

## 你要偵測的，是什麼？

![HC-SR501 PIR 感測器模組正面，白色圓頂為菲涅爾透鏡](https://upload.wikimedia.org/wikipedia/commons/6/6f/PIR_Motion_Sensor-Sensinova_%28SN-PR11%29.png "PIR 感測器模組")
*圖片來源：Wikimedia Commons / CC BY-SA 4.0*

在設計任何偵測系統之前，需要先想清楚一件事：你要偵測的，是什麼？

PIR 感測器要解決的問題很具體：這個空間裡，有沒有溫熱的東西在移動？

不是「這個人站在哪裡」，不是「有幾個人」，也不是「那個人走了多快」。就是，有，還是沒有。

這個邊界搞清楚，你就知道什麼情況適合用它，什麼情況用它只會讓你做白工。需要知道精確位置的，用超聲波。需要穿牆偵測的，用微波雷達。只需要知道有沒有人在動，用 PIR。

---

## 它感知的是你身體的熱

PIR 的全名是 Passive Infrared，被動紅外線。

被動，是說它不主動發出任何東西。不像超聲波感測器那樣發射聲波，也不像雷達那樣發射電磁波。它只是靜靜接收。

接收的是什麼？是你身體輻射的熱，也就是紅外線。

人體溫度大約 36 到 37°C，這個溫度會持續向外輻射波長約 9 到 10 微米的紅外線。你眼睛看不到，但 PIR 的感測元件對這個波長範圍非常敏感。

為什麼要用紅外線，不用其他方式？因為在室內環境裡，人體輻射的這個波長非常穩定。白天黑夜都一樣，不管燈怎麼開關，你的熱輻射就在那裡。這讓 PIR 在完全全暗的走廊裡，也能正常偵測到人。

## 感測的不是熱，是熱在移動

![PIR 感測器電路板內部，可以看到感測元件與訊號處理電路](https://upload.wikimedia.org/wikipedia/commons/7/7d/Toom_motion_detector_-_board_-_Pollin_PIR_D203S-2662.jpg "PIR 感測器電路板內部")
*圖片來源：Wikimedia Commons / CC BY-SA 4.0*

PIR 感測元件裡面有兩個並排的熱電堆（Pyroelectric Element）。

你可以這樣想像：兩個人並排坐著，各自感受周圍的溫度。沒有熱源移動的時候，兩個人感受到的溫度差不多，沒什麼事。

但是當你走過去，你的熱先掃到左邊那個人，再掃到右邊那個人。兩個人感受到的溫度出現了先後差異，這個差異產生一個電壓變化，感測器輸出 HIGH，告訴後端電路「有東西動了」。

等你停下來站著不動，兩個人都感受到你的熱了，差異消失，訊號回到 LOW。

所以 PIR 感測的不是「有熱在那裡」，是「熱在移動」這件事。

這就解釋了一個很常見的問題：你在走廊站著不動，幾秒後燈滅了。不是感測器壞了，是你靜止之後它真的感覺不到你了。

感測元件前面那顆白色圓頂是菲涅爾透鏡（Fresnel Lens）。它把來自各個方向的紅外線聚焦到感測元件上，讓一個小感測器可以覆蓋 120 度的偵測角度，感測距離最遠可以到 7 公尺。

## 跟其他方案比起來，PIR 適合在什麼情況用？

| 方案 | 感知原理 | 最適合的情況 | 功耗 | 成本 |
|------|----------|-------------|------|------|
| PIR | 被動接收紅外線熱輻射 | 人體移動偵測、自動燈光、低功耗裝置 | 極低 | 低 |
| 超聲波 | 主動發射聲波，量回波時間 | 測距、障礙物偵測、機器人導航 | 低 | 低 |
| 微波雷達 | 多普勒效應偵測移動 | 穿牆偵測、細微動作感測（如呼吸） | 中 | 中 |
| 相機 + AI | 影像識別 | 人臉辨識、行為分析、複雜場景理解 | 高 | 高 |

PIR 最大的優勢是被動和省電。它不需要持續發射任何東西，待機功耗幾乎可以忽略。這就是為什麼電池驅動的保全感測器可以不換電池撐好幾年。

限制也很明確：只能知道有沒有，不能知道在哪裡；靜止物體偵測不到；無法穿透牆壁。如果你的應用需要精確位置、穿牆偵測、或識別具體是什麼東西，那 PIR 做不到，換其他方案。

## 實作：HC-SR501 加 Arduino，十分鐘做出動作偵測原型

![走廊裝有 PIR 感測器的牆壁開關，展示實際應用場景](https://upload.wikimedia.org/wikipedia/commons/c/cd/Light_switch_with_passive_infrared_sensor.jpg "PIR 感測器應用於走廊燈控制")
*圖片來源：Wikimedia Commons / CC BY-SA 4.0*

市面上最常用的 PIR 模組是 HC-SR501（[Amazon 連結]）。蝦皮或露天搜尋直接找得到，一顆大概 NT$20 到 50。

模組上有兩個旋鈕，可以用螺絲起子調整：

- 左邊旋鈕：靈敏度，控制偵測距離，調整範圍大約是 3 公尺到 7 公尺
- 右邊旋鈕：延遲時間，控制觸發後訊號維持多久，幾秒到幾分鐘都可以調

接線方式：

| HC-SR501 腳位 | Arduino 腳位 |
|--------------|-------------|
| VCC | 5V |
| GND | GND |
| OUT | D2 |

程式碼：

```cpp
const int PIR_PIN = 2;
const int LED_PIN = 13;

void setup() {
  pinMode(PIR_PIN, INPUT);
  pinMode(LED_PIN, OUTPUT);
  Serial.begin(9600);

  // HC-SR501 上電後需要約 30 秒讓感測元件校正背景紅外線
  // 這段時間內會反覆誤觸發，是正常現象，等它穩定就好
  delay(30000);
  Serial.println("就緒，開始偵測");
}

void loop() {
  if (digitalRead(PIR_PIN) == HIGH) {
    digitalWrite(LED_PIN, HIGH);
    Serial.println("偵測到動作");
  } else {
    digitalWrite(LED_PIN, LOW);
  }
  delay(100);
}
```

一個常見的問題：上電之後的第一個 30 到 60 秒，LED 會一直亂閃，感覺好像程式有問題。這其實是 HC-SR501 在做自我校正，它需要先「記住」目前環境的紅外線背景值，才能正確判斷什麼是真正的動作。`delay(30000)` 就是讓程式等這段時間過去再開始偵測。

上傳程式後，等 30 秒，然後在感測器前面揮動手，Serial Monitor 應該會出現「偵測到動作」。

## 你做出來的這個東西，還可以往哪裡走

你現在做出來的，是一個 LED 對你的走動有反應的裝置。

把觸發邏輯換掉，讓它觸發的不是 LED，而是相機拍照、記錄時間戳記、或者發送一個通知。整件事的感覺就完全不一樣了。

野外用來拍攝野生動物的相機陷阱（Camera Trap），核心邏輯和你現在做的這個東西是一樣的。PIR 偵測到熱源移動，相機觸發拍照，把動物走過的那一刻記錄下來。
