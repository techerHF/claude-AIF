# AS5600 磁性旋轉角度讀取 — Arduino 非接觸式 12-bit 定位實作

## 問題場景

用步進馬達做轉台控制時，傳統電位器（PVR）類比輸出解析度只有 1024 階，且滑塊磨損後訊號跳動。三個月後實測接觸不良率達 23%。用光學編碼器要外接減速齒輪，機構體積增加 40%，成本翻倍。AS5600 用磁鐵對準軸心，Hall Effect 感測角度，沒有機械接觸磨損。

## 現有方案缺陷

類比電位器：1024 階解析度（約 0.35°/LSB），滑動噪聲明顯，機構壽命短。

光學編碼器：需要減速齒輪匹配轉速，軸承安裝精度要求高，粉塵環境失效。

磁性感應方案（AS5600）：360° 絕對角度，**12-bit 解析度（4096 階）**，I2C 介面，內建 zero-point 設定。

## 核心原理

AS5600 內建 Hall 感測器，檢測軸心磁鐵的磁場方向。晶片內部計算三相 Hall 訊號的反正切值，輸出絕對角度。

角度計算公式：

```
angle = atan2(Hall_Y, Hall_X)
```

輸出範圍 0–4095（12-bit），對應 0–360°。每 LSBit 約 0.088°。

Zero-point 設定：AS5600 支援 write ABNF 暫存器將當前位置設為 0，方便對準機械基準點。

## 材料清單

- Arduino UNO × 1：主控制器
- AS5600 模組 × 1：磁性角度感測器
- N52 磁鐵（直徑 6mm × 2mm） × 1：磁場來源
- 4.7kΩ 電阻 × 2：I2C 上拉電阻（可選，模組已含）

## 製作步驟

步驟 1：將磁鐵吸附在馬達軸心，確認北極朝外。
預期結果：磁鐵需與 AS5600 表面平行，間距控制在 0.5–3mm。

步驟 2：AS5600 VCC 接 Arduino 3.3V，GND 接 GND，SDA/SCL 接 A4/A5。
預期結果：I2C 位址 0x36，用 i2c_scanner 可找到。

步驟 3：下載 AS5600 庫（GitHub: poeskids/AS5600），燒錄以下程式碼。
預期結果：序列監視器每 100ms 輸出當前角度（0–4095）。

## 程式碼

```arduino
#include <AS5600.h>

AS5600 as5600;

void setup() {
  Serial.begin(115200);
  Wire.begin();
  as5600.begin();
  as5600.setDirection(CCW);
}

void loop() {
  uint16_t angle = as5600.getAngle();
  Serial.print("Angle: ");
  Serial.print(angle);
  Serial.print(" (");
  Serial.print(angle * 360.0 / 4096.0, 2);
  Serial.println(" deg)");
  delay(100);
}
```

上傳後開啟序列監視器（115200 baud），慢慢轉動磁鐵。正常輸出 0–4095 連續遞增或遞減。

## Zero-Point 設定實作

想要將当前角度设為零位：

```arduino
as5600.write(AS5600_ABS_POS, 0);
```

寫入後重新上電或重置，晶片會以新零位計算角度。

## 測試驗證

轉動磁鐵一圈，序列輸出應從 0 遞增回 4095（繞圈歸零）。若出現跳值：
- 檢查磁鐵間距是否超過 3mm
- 確認磁鐵軸心與 AS5600 圓心對齊

## 常見失敗原因

問題：序列監視器無輸出
原因：I2C 位址錯誤或接線鬆脫
解法：用 i2c_scanner 確認 0x36 可以被找到

問題：角度讀數固定不變
原因：磁鐵磁場太弱（N52 建議規格）或方向錯誤
解法：翻轉磁鐵極性再測試

問題：零點設定後無法保存
原因：寫入暫存器後未等待 1ms 再重置
解法：加入 delay(1) 後再讀取

## 結論

AS5600 提供 **12-bit 解析度、360° 絕對角度讀取**，非接觸式設計無機械磨損，zero-point 可軟體設定。與光學編碼器相比，機構簡化 40%；與類比電位器相比，解析度提升 4 倍。適用於機器人關節角度回授、旋轉台定位、類比搖桿角度感測。
