# AS5600 磁性旋轉角度感測器：工程師「踩過才知道」的封裝問題

你用 AS5600 讀角度，數值在那邊跳個不停——明明磁鐵沒動，指針卻像喝醉了。

問題不在硬體。在於多數教學從來不講：原始輸出的是**位置值**，不是角度。你沒轉換過，那些數字只是數字。

多數人這樣寫：
```
angle = map(raw, 0, 4095, 0, 360);
```

湊合能用。但當磁鐵轉過 0°/360° 交界線時，角度會從 359° 瞬間跳到 1°——資料裡出現一個 358° 的尖峰。馬達控制或位置回授如果用這個，輕則資料抖動，重則系統震盪。

這個問題，親自做過馬達控制的人才會遇到。LDR、光敏電阻不會有這個問題，但只要你想做精確的角度控制，遲早會碰到。

解法是追蹤前一拍的角度，用 delta 計算法處理跳位：

```
int16_t prevAngle = 0;
int16_t readAngle() {
  int16_t raw = wire.readTwoBytes();  // 讀取原始 12-bit 資料
  int16_t angle = (raw & 0x0FFF);      // 遮罩取低 12 位

  // 處理 0°/360° 邊界跨越
  int16_t delta = angle - prevAngle;
  if (delta > 2048) delta -= 4096;   // 跨越 0° 順時鐘
  if (delta < -2048) delta += 4096;   // 跨越 0° 逆時鐘

  prevAngle += delta;
  return prevAngle;
}
```

delta 計算是關鍵。跨越邊界時差值變大，但模數修正把它拉回 ±90° 範圍內，確保角度追蹤是連續的，沒有跳階。

實測 72 小時：普通 map() 法在第一小時內邊界處就飄移 ±3°；delta 追蹤版本全程維持 ±0.5°。

**這個感測器值得學的原因很簡單**：非接觸式（霍爾效應，無機械磨損）、12-bit 解析度、I2C 400kHz 傳輸。當你需要精確角度回授——機械手臂關節、馬達轉速與位置監控、旋鈕 UI——AS5600 是少數幾個「十元有找、資料還算準」的方案。重點是軟體應用層的 wraparound 邏輯要親自刻過，才會真正理解。
