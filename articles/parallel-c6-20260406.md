# MCP4725 類比波形產生器 - Arduino 12位元 DAC，4096階解析度，無需低通濾波

## 問題場景

你需要用 Arduino 輸出可變電壓類比訊號，驅動類比式感測器或製作簡單訊號產生器。 PWM 輸出需要外接低通濾波電路，且解析度只有 256 階（8 位元）。你聽說 MCP4725 可以直接輸出真實類比電壓，但查了範例程式却不知道如何產生波形、更新速率能多快、以及查表法要怎麼實作。

## 現有方案的問題

PWM 類比輸出有兩個根本缺陷。第一，解析度受限於計時器位元數，Arduino UNO 的 PWM 只有 **256 階**，無法满足 12 位元（**4096 階**）需求。第二，PWM 輸出是方波，必須外接 **RC 低通濾波器**才能轉換為近似直流電壓，濾波器會拖慢響應速度，高頻波形更無法忠實重現。

MCP4725 是 **12 位元 I2C DAC**，內建 **4.096V 參考電壓**，輸出解析度為 **4096 階**，無需濾波電路即可輸出直流或低頻交流波形。

## 核心原理

MCP4725 透過 I2C 介面接收 16 位元命令（12 位元資料 + 4 位元指令），輸出 0-4.096V 類比電壓。理論更新速率取決於 I2C 時脈：標準模式 **100 kHz** 下，最大更新率約 **6.25 kSPS**；快速模式 **400 kHz** 下，可達 **25 kSPS**。實際值低於理論值，因為每次寫入需要傳輸 3 位元組（控制位元組 + 高位元組 + 低位元組）。

波形生成採用預先計算的 **waveforms 查表法**。將一個週期的波形樣點預先存入陣列，輪流讀取即可輸出連續波形。假設取樣 **360 點**，而 I2C 更新率為 **10 kSPS**，可產生約 **27.8 Hz** 的正弦波。

```
公式：輸出頻率 = I2C更新率 / 取樣點數
```

## 材料清單

- Arduino UNO × 1：主控制器
- MCP4725（Adafruit MCP4725）× 1：12位元 DAC
- 示波器或萬用電表 × 1：驗證輸出波形（選配）
- 10kΩ 電阻 × 1：輸出負載（可選）

## 製作步驟

步驟1：連接線路
預期結果：SDA→A4，SCL→A5，VCC→3.3V，GND→GND
注意事項：MCP4725 支援 3.3V 或 5V 供電，I2C 位址為 **0x62**（預設）

步驟2：安裝 MCP4725 函式庫
預期結果：Sketch > Include Library > Manage Libraries，找到「Adafruit MCP4725」
注意事項：亦可使用 Wire 函式庫直接存取 I2C 暫存器

步驟3：定義 waveforms 查表陣列
預期結果：360 點正弦波表格，值域 0-4095（12位元）
注意事項：查表可為正弦波、方波、三角波或任意自定義波形

步驟4：主迴圈輪流查表輸出
預期結果：連續輸出波形，頻率由 I2C 更新速率與取樣點數決定
注意事項：I2C 延遲是主要瓶頸，不建議在主迴圈中使用 delay()

## 測試驗證

驗證方法：用示波器測量 MCP4725 的輸出腳，確認波形形狀與頻率。
正常值：正弦波幅度 **0-4.096V**，頻率約 **27.8 Hz**（10kSPS / 360點）
異常值：波形失真或頻率不符，檢查 I2C 連線或增速時脈至 400kHz

## 常見失敗原因

問題：無波形輸出
原因：I2C 接線錯誤（SDA/SCL 順序）
解法：確認 SDA→A4，SCL→A5，或使用 Mega 的 **20/21** 腳位

問題：波形頻率低於預期
原因：I2C 時脈為預設 100kHz，而非 400kHz
解法：加入 `Wire.setClock(400000)` 提速

問題：輸出電壓範圍不符
原因：未使用內建 4.096V 參考（第二參數設為 false）
解法：確認 dac.setVoltage() 第二參數為 false，或外部供電時需自行計算

## 延伸方向

此基礎可延伸為任意波形產生器（AWG）：替換 waveforms 表格為自定義波形陣列，支援方波、三角波或心電圖波形。亦可結合 ESP32 的 I2S 或 DMA 功能，突破 I2C 頻寬限制，輸出更高頻波形（達 100kHz 以上）。
