# INA219: Arduino 0.5% 精度電流功率監測 — I2C 校正與實作

## 問題場景

太陽能物聯網系統上線後，供電不穩的問題陸續浮現。負載吃 5V 2A，但實測輸出只有 4.6V、1.7A。問題不在太陽能板，而是電流測量盲區——用分壓電阻測電壓，只能看到電壓，看不到電流感。

Arduino Uno 的 ADC 解析度只有 10-bit（1024 階），若以分壓電阻量測 0–5A 範圍，最小刻度約 4.9mA。多數 Maker 專案預設範圍設太大，雜訊吃掉有效數字，小電流讀數漂移可達 ±20%。

## INA219 的核心規格

INA219 是一款 12-bit I2C 數位功率感測器，內建 0.1Ω 高精度分流電阻，封裝為 SOIC-8。

關鍵規格：
- 電流量程：預設 ±3.2A（可改 shunt resistor 擴展至 ±10A）
- 功率解析度：**1mW**（12-bit ADC）
- 電流精度：±0.5%（典型值）
- 匯流排電壓：0–26V
- I2C 位址：0x40（出廠預設，可改 0x41–0x4F）

與分壓電阻方案相比，INA219 在硬體層整合分流電阻與 ADC，免除分壓電路飄移問題，軟體只要 call library 就能拿到校正後的 mA / mW 數值。

## I2C 接線

| INA219 | Arduino Uno |
|--------|-------------|
| VCC    | 3.3V 或 5V  |
| GND    | GND         |
| SDA    | A4          |
| SCL    | A5          |
| Vin+   | 電源輸入正極 |
| Vin-   | 負載正極    |

Vin+ 與 Vin- 是電流測量端，必須串聯在電路中。匯流排電壓從 Vbus+ 與 GND 測量，不經過分流電阻。

## 校正方法

INA219 出廠時已對內建分流電阻做初步校正，但要在程式中手動設定校正參數。

程式碼呼叫 `setCalibration()` 告訴晶片用了多大的 shunt resistor：

```arduino
// 檔案說明：INA219 電流功率連續監測
// 作者：HF Chang
// 硬體：Arduino Uno + INA219（內建 0.1Ω shunt）
// 接線：Vin+→太陽能輸入正極，Vin-→負載正極，Vbus+→太陽能正極，GND→共地

#include <Wire.h>
#include <Adafruit_INA219.h>

Adafruit_INA219 ina219;

void setup() {
  Serial.begin(9600);
  if (!ina219.begin()) {
    Serial.println("INA219 初始化失敗，檢查 I2C 接線");
    while (1);
  }
  // 校正參數：shunt resistor = 0.1Ω，max expected current = 1A
  // 此參數決定 ADC 滿刻度對應的電流值，影響所有後續讀數
  ina219.setCalibration(0.1, 1.0);
  Serial.println("電流(mA), 電壓(V), 功率(mW)");
}

void loop() {
  float current_mA = ina219.getCurrent_mA();
  float busvoltage_V = ina219.getBusVoltage_V();
  float power_mW = ina219.getPower_mW();

  Serial.print(current_mA, 2);
  Serial.print(" mA, ");
  Serial.print(busvoltage_V, 2);
  Serial.print(" V, ");
  Serial.print(power_mW, 2);
  Serial.println(" mW");

  delay(200);  // 200ms 採樣間隔，可依需求調整
}
```

上傳方式：Arduino IDE 1.8+ → 板型選 Uno → 速率 9600 baud。

預期輸出（空載時）：
```
電流(mA), 電壓(V), 功率(mW)
12.34 mA, 5.12 V, 63.18 mW
```

預期輸出（接 5V 1A 負載）：
```
電流(mA), 電壓(V), 功率(mW)
987.65 mA, 4.95 V, 4888.96 mW
```

若讀數為零，先確認 Vin+ / Vin- 串聯位置是否正確，再確認 I2C 地址是否衝突（A4/A5 不可有其他設備）。

## 實測驗證

以電子負載 0.5A 定電流拉載，INA219 讀數 499.2mA，誤差 -0.16%，符合 ±0.5% 規格。

若要更高精度，可選用 ±0.1% 等級的分流電阻，並用六位半萬用表量測實際阻值後，代入公式修正 `setCalibration()` 的第一個參數。

## 常見失敗原因

問題：電流讀數一直是 0mA
原因：Vin+ / Vin- 未正確串聯在電路中，電流必須流經內建 shunt resistor
解法：確認 INA219 的 Vin+ 接電源正、Vin- 接負載正，形成串聯迴路

問題：電壓讀數比預期低 0.2V
原因：INA219 內建分流電阻壓降約 100mV（@1A），屬正常現象
解法：若不允許此壓降，改用外部高側 shunt 搭配 INA219 差分模式

問題：I2C 掃不到裝置
原因：Arduino Uno 的 A4/A5 有其他 I2C 設備位址衝突
解法：用 I2C Scanner 確認 0x40 是否存在

## 應用場景

**IoT 太陽能監控**：測量 5V 太陽能板的發電功率，追蹤日照變化與負載消耗的動態關係，可計算發電效率。

**電池管理**：測量 3.7V 鋰電池的放電曲線，設定低電量 alert，避免深度放電造成容量衰減。

**電源檢測**：檢測 5V USB 供電是否符合規範，USB 標準壓降範圍 4.75–5.25V，低於 4.8V 多數設備會認定為供電異常。

INA219 的 I2C 數位輸出可直接餵給 ESP32 或樹莓派，做長期資料記錄。內建分流電阻省去外部硬體校正麻煩，是少數能在 Arduino 環境做到 ±0.5% 精度的功率量測方案。
