# ADXL345 衝擊偵測靈敏度設定：THRESH_TAP 62.5 mg/LSB 詳解

## 問題場景

你在製作一款手環，設計「跌倒偵測」功能。ADXL345 設定 THRESH_TAP = 30 以為等於 30g，實際上晶片手勢偵測完全沒反應。問題出在 ADXL345 的 THRESH_TAP 單位是 **62.5 mg/LSB**，30 代表的是 30 × 62.5 = 1875 mg（約 0.19g），遠低於你的預期值。

這個單位轉換錯誤是 DIY 玩家最常見的踩坑點。另一個常被忽略的問題是輪詢（polling）模式讓 MCU 持續喚醒，功耗高達 130 μA。本篇說明如何用 interrupt-based 偵測，功耗降至 0.1 μA，同時精準設定靈敏度。

## 現有方案的問題

多數教學使用輪詢讀取 XYZ 原始值，再自行寫門檻判斷。這個做法有兩個缺陷：第一，MCU 每秒喚醒數百次，電流消耗 130 μA，無法適用於穿戴式电池供电设备。第二，自由落體偵測（Free-Fall）需要及時響應，輪詢延遲可達 10–20 ms，可能漏報。

 Interrupt 模式的優點是晶片內部自行偵測，符合條件才喚醒 MCU，功耗降至 0.1 μA，響應時間小於 1 ms。

## 核心原理

ADXL345 內建兩級偵測機制：TAP（輕擊）與 FF（Free-Fall）。兩者共享相同的 threshold 寄存器 THRESH_TAP，但偵測邏輯不同。

**THRESH_TAP 轉換公式：**

```
實際門檻值(g) = THRESH_TAP × 62.5 mg/LSB / 1000
```

若要偵測 3g 以上的衝擊，THRESH_TAP 設定為：

```
THRESH_TAP = 3g / 0.0625g = 48
```

**TAP 偵測條件：** 在 DUR 時間窗口內，加速度超過 THRESH_TAP 且低於 2×THRESH_TAP，視為單擊。超過 2×THRESH_TAP 的訊號視為雙擊。

**FF 偵測條件：** 加速度持續低於 FF_THRESH（建議 300–600 mg）且時間在 FF_TIME 窗口內（建議 100–500 ms）。

## 材料清單

- Arduino UNO × 1：主控制器
- ADXL345（SparkFun 模組）× 1：I2C 介面，加速規
- 3.7V鋰電池 × 1：供電（可選）
- 杜邦線 10cm × 4：連接 I2C（SDA、SCL、VCC、GND）

## 製作步驟

**步驟1：接線**
ADXL345 VCC 接 3.3V（勿接 5V，會損壞晶片），SDA 接 A4，SCL 接 A5，GND 接 GND。

**步驟2：設定interrupt針腳**
將 INT1 針腳接到 Arduino D2（interrupt 針腳），用於接收 ADXL345 的中斷信號。

**步驟3：燒錄程式（見下一節）**

**預期結果：** 序列監視器每1秒輸出加速度值。輕敲感測器時，螢幕印出 "TAP Detected"。

## 程式碼

```arduino
#include <Wire.h>

#define ADXL345_ADDR 0x53
#define THRESH_TAP_REG 0x1D
#define TAP_AXES_REG 0x2A
#define INT_ENABLE_REG 0x2E
#define INT_MAP_REG 0x2F
#define POWER_CTL_REG 0x2D
#define DUR_REG 0x21
#define FF_THRESH_REG 0x28
#define FF_TIME_REG 0x29

// 設定門檻值：3g → THRESH_TAP = 3000 / 62.5 = 48
const unsigned char THRESH_TAP_VAL = 48;
const unsigned char DUR_VAL = 0x20;        // 320 μs 窗口
const unsigned char TAP_AXES_VAL = 0x07;   // XYZ 全部啟用
const unsigned char FF_THRESH_VAL = 30;    // 30 × 62.5 mg = 1.875g
const unsigned char FF_TIME_VAL = 0x14;    // 200 ms 窗口

void setup() {
  Serial.begin(115200);
  Wire.begin();

  // 進入測量模式
  Wire.beginTransmission(ADXL345_ADDR);
  Wire.write(POWER_CTL_REG);
  Wire.write(0x08);        // Measure = 1
  Wire.endTransmission();

  // 設定 TAP 門檻
  Wire.beginTransmission(ADXL345_ADDR);
  Wire.write(THRESH_TAP_REG);
  Wire.write(THRESH_TAP_VAL);
  Wire.endTransmission();

  // 設定 TAP 持續時間
  Wire.beginTransmission(ADXL345_ADDR);
  Wire.write(DUR_REG);
  Wire.write(DUR_VAL);
  Wire.endTransmission();

  // 啟用 XYZ TAP 偵測
  Wire.beginTransmission(ADXL345_ADDR);
  Wire.write(TAP_AXES_REG);
  Wire.write(TAP_AXES_VAL);
  Wire.endTransmission();

  // 設定 Free-Fall 門檻與時間
  Wire.beginTransmission(ADXL345_ADDR);
  Wire.write(FF_THRESH_REG);
  Wire.write(FF_THRESH_VAL);
  Wire.endTransmission();

  Wire.beginTransmission(ADXL345_ADDR);
  Wire.write(FF_TIME_REG);
  Wire.write(FF_TIME_VAL);
  Wire.endTransmission();

  // 啟用 TAP + Free-Fall interrupt，映射至 INT1
  Wire.beginTransmission(ADXL345_ADDR);
  Wire.write(INT_ENABLE_REG);
  Wire.write(0x03);        // BIT0 = TAP, BIT1 = FREE-FALL
  Wire.endTransmission();

  // 附加 interrupt 處理函式
  attachInterrupt(digitalPinToInterrupt(2), tapISR, RISING);

  Serial.println("ADXL345 Interrupt Mode Ready");
  Serial.print("TAP threshold: ");
  Serial.print(THRESH_TAP_VAL * 62.5 / 1000.0);
  Serial.println(" g");
}

void loop() {
  Wire.beginTransmission(ADXL345_ADDR);
  Wire.write(0x32);  // X0 LSB
  Wire.endTransmission(false);
  Wire.requestFrom(ADXL345_ADDR, 6);

  int16_t x = Wire.read() | (Wire.read() << 8);
  int16_t y = Wire.read() | (Wire.read() << 8);
  int16_t z = Wire.read() | (Wire.read() << 8);

  // 轉換為 g 值（14-bit 解析度，SCALE = 3.9 mg/LSB）
  float xg = x * 0.0039;
  float yg = y * 0.0039;
  float zg = z * 0.0039;

  Serial.print("X="); Serial.print(xg, 3);
  Serial.print("g Y="); Serial.print(yg, 3);
  Serial.print("g Z="); Serial.print(zg, 3);
  Serial.println("g");

  delay(100);
}

void tapISR() {
  // 讀取 INT_SOURCE 確認哪個中斷觸發
  Wire.beginTransmission(ADXL345_ADDR);
  Wire.write(0x30);  // INT_SOURCE
  Wire.endTransmission(false);
  Wire.requestFrom(ADXL345_ADDR, 1);

  byte intSource = Wire.read();

  if (intSource & 0x03) {
    Serial.println("TAP Detected!");
  }
  if (intSource & 0x02) {
    Serial.println("FREE-FALL Detected!");
  }
}
```

**上傳方式：** Arduino IDE 選擇「Arduino UNO」板，燒錄至板子。開啟序列監視器， baud rate 設 115200。

**預期輸出：** 安靜狀態下每秒印出三軸 g 值（約 0g, 0g, 1g，因為重力）。輕敲感測器時，序列監視器印出 "TAP Detected!"。

## 測試驗證

正常值：Z 軸靜止約 1.0g（重力），X/Y 軸接近 0g。
敲擊測試：用手指輕敲模組，序列監視器顯示 "TAP Detected!"。
自由落體測試：將模組自由落下（高於 20cm），顯示 "FREE-FALL Detected!"。

## 常見失敗原因

**問題：無論怎麼敲都沒反應**
原因：THRESH_TAP 設太高或太低。太高則靈敏度不足，太低會把環境震動當成 TAP。
解法：用公式反推。目標 2g 靈敏度，THRESH_TAP = 2 / 0.0625 = 32。

**問題：Free-Fall 一直誤觸發**
原因：FF_THRESH 設太低（低於 300 mg）或 FF_TIME 太長（超過 500 ms）。
解法：FF_THRESH 設為 30–50（1.9–3.1g），FF_TIME 設為 0x14（200 ms）。

**問題：序列監視器無輸出**
原因：I2C 地址錯誤（ADXL345 預設 0x53，但部分模組是 0x1D）。
解法：執行 I2C scanner 確認地址。
