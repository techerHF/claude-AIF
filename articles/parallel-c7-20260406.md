# MAX6675 K型熱電偶高溫量測：內建冷接點補償，輸出直接是溫度

## 問題場景

窯爐或工業烤箱需要監測 800°C 以上的內部溫度工程師直覺想到用 K 型熱電偶。但類比電壓讀回來往往差異極大——同樣的感測器，不同室溫下量到的電壓差異可達數 mV。這不是感測器損壞，而是「冷接點補償」被忽略了。

## 現有方案缺陷

常見錯誤做法是直接用 ADC 讀取熱電偶產生的 Seebeck 電壓。K 型熱電偶在 800°C 時約產生 33 mV，但室溫 25°C 也貢獻約 1 mV。如果不做補償，實際讀值會比真實溫度低約 25°C。另一個常見問題是熱電偶輸出是μV/°C 等級（K 型約 41 μV/°C），一般 Arduino 的 10-bit ADC 解析度僅 4.9 mV，根本無法精準量測。

## 核心原理

MAX6675 解決了兩個問題。第一，它是一顆專用冷接點補償 IC，內建室溫參考端感測器，會自動將測量端的熱電勢加上冷端的反向補償電壓，最終輸出即為實際溫度。第二，它內建 12-bit ADC，將類比熱電勢數位化後以 SPI 介面輸出，解析度 **0.25°C**（12-bit / 1024 = 0.25）。

輸出格式：SO 腳位的 16-bit 資料中，D15 為符號位（0 = 正），D14–D3 為溫度值（12-bit），乘以 0.25°C 即為實際溫度。D2 為感測器斷線偵測旗標。

```arduino
// 檔案說明：MAX6675 K型熱電偶溫度量測
// 作者：HF Chang
// 硬體：Arduino UNO、MAX6675、K型熱電偶
// 接線：SO→D12、CS→D10、CLK→D13

const int CS_PIN = 10;   // CS (chip select)
const int SO_PIN = 12;   // SO (serial out)
const int CLK_PIN = 13;  // CLK (clock)

void setup() {
  Serial.begin(9600);
  pinMode(CS_PIN, OUTPUT);
  pinMode(SO_PIN, INPUT);
  pinMode(CLK_PIN, OUTPUT);
  digitalWrite(CS_PIN, HIGH);  // 閒置時拉高 CS
}

void loop() {
  float temp = readMax6675();
  Serial.print("溫度：");
  Serial.print(temp, 2);
  Serial.println(" °C");
  delay(500);
}

float readMax6675() {
  // 讀取 16-bit 資料
  digitalWrite(CS_PIN, LOW);   // 開始讀取，拉低 CS
  delayMicroseconds(10);       // tCC（CS 到第一個 CLK 的最小時間）

  unsigned int result = 0;
  for (int i = 15; i >= 0; i--) {
    digitalWrite(CLK_PIN, HIGH);
    delayMicroseconds(1);
    bitWrite(result, i, digitalRead(SO_PIN));
    digitalWrite(CLK_PIN, LOW);
    delayMicroseconds(1);
  }

  digitalWrite(CS_PIN, HIGH);  // 讀取完成，拉高 CS

  // D15=0（永遠為0）、D2=斷線偵測
  if (bitRead(result, 2)) {
    Serial.println("錯誤：熱電偶斷線");
    return -1;
  }

  // D14–D3 為溫度值（12-bit），解析度 0.25°C
  result >>= 3;               // 移除 D2、D1、D0
  return result * 0.25;       // 單位：°C
}
```

上傳方式：Arduino IDE 1.8 以上，選擇「Arduino UNO」板，Serial Monitor 設定 9600 baud。正常輸出約 25°C（室溫），手握熱電偶接點可見數值上升。

## 材料清單

- Arduino UNO × 1：主控制器
- MAX6675 模組 × 1：含冷接點補償 IC
- K 型熱電偶（不銹鋼護套）× 1：量測端，最高 1023.75°C
- 杜邦線（公對母）× 4：連接模組與 Arduino

## 常見失敗原因

**問題：讀值一直是 -1 或 0**
原因：熱電偶接點斷線，D2 旗標被設定。
解法：檢查熱電偶焊接點或更換新的感測器。

**問題：室溫下讀值比實際低約 20–30°C**
原因：使用一般 ADC 直接讀取熱電偶電壓，沒有冷接點補償。
解法：改用 MAX6675，輸出已包含補償值。

**問題：SPI 讀取數值跳動**
原因：CLK 頻率過高（MAX6675 最大 4.3 MHz）或雜訊干擾。
解法：在 SO 和 GND 之間並聯 100 nF 電容，或降低 CLK 頻率。

## 驗證方式

將 K 型熱電偶放入冰水混合物（0°C）中，讀值應在 -1 至 +1°C 範圍內。放入沸水（100°C，海拔修正後）中，讀值應在 99–101°C 範圍內。若偏差超過 2°C，檢查熱電偶是否老化。

## 延伸方向

若需要多點溫度量測，可串聯多顆 MAX6675，透過不同 CS 腳位分別讀取。若量測距離超過 1 公尺，建議使用延長線型熱電偶並注意屏蔽，以避免 EMI 干擾。
