# 用 5 個彎曲感測器做手套控制器：ESP32 讀取 0–90° 彎曲角度，延遲低於 20 ms

## 問題場景

你想讓 ESP32 專案「看懂」手指動作，但現有方案不是太貴就是太複雜。
Leap Motion 要 80 美元，DataGlove 要幾千元。
最直接的方案：每根手指裝一個彎曲感測器，讀取彎曲角度，成本 **200 元**。

## 現有方案的問題

市售彎曲感測器（如 SpectraSymbol 2.2英寸款）單顆電阻範圍 **8.0–45.0 kΩ**。
問題在於：這個範圍個體差異大，同型號兩顆感測器在同樣彎曲角度，電阻差異可達 **15%**。
如果直接用電阻值判斷角度，不同手套的讀值完全不同，無法移植程式碼。

解法不是換感測器，而是在程式裡做個人校正（calibration），讓讀值歸一化到 **0–90° 範圍**。

## 核心原理

彎曲感測器本質是可變電阻：碳膜基材彎曲時，電阻隨彎曲角度增加。
搭配固定電阻 **47 kΩ** 組成分壓電路，輸出電壓：

**V_out = V_cc × R_fixed / (R_fixed + R_flex)**

ESP32 的 ADC 解析度為 **12-bit**，輸出 0–4095，對應 0–3.3V。
未彎曲時 V_out 約 **1.8V**（ADC ≈ 2232），完全彎曲時 V_out 約 **0.9V**（ADC ≈ 1116）。
線性映射即可得到 0–90° 的角度估算，誤差在 **±5°** 以內。

## 材料清單

- ESP32 DevKit V1 × 1：主控制器，5 個類比輸入（GPIO34-39）
- 彎曲感測器 2.2英寸 × 5：各對應一根手指
- 47 kΩ 電阻 × 5：分壓電路
- 手套（薄款）× 1：固定感測器用
- 布膠帶 × 適量：感測器固定
- 跳線 × 15 條：接線

材料總成本約 **185 元**（不含 ESP32）。

## 製作步驟

**步驟1：接線（每根手指相同）**
彎曲感測器一端 → 3.3V；另一端 → 47 kΩ 電阻 → GND。
感測器和電阻的接合點 → ESP32 類比輸入（GPIO34–38）。
預期結果：Serial Monitor 顯示各腳位 ADC 數值，手指伸直約 2000–2400。

**步驟2：個人校正（關鍵步驟）**
上傳校正程式，手指完全伸直，按 Enter 記錄各腳位的 `flat_value`。
接著手指完全彎曲（握拳），按 Enter 記錄 `bent_value`。
這兩個數值儲存為常數，後續映射用。
注意：每次換新手套或感測器位置改變都需要重新校正。

**步驟3：角度映射**
用 Arduino 的 `map()` 函式將 ADC 值映射到 0–90°。

## 程式碼

```arduino
// 5指彎曲感測手套控制器
// 作者：HF Chang
// 硬體：ESP32 DevKit V1 + 彎曲感測器 × 5 + 47kΩ電阻 × 5
// 接線：各感測器分壓點 → GPIO34(拇指), 35(食指), 36(中指), 39(無名), 34(小指)
// 注意：GPIO34-39 為輸入專用，不能輸出

// 校正數值（執行 calibration 後填入）
const int FLAT[5]  = {2300, 2250, 2180, 2310, 2260};  // 伸直時 ADC
const int BENT[5]  = {1100, 1050, 1020, 1080, 1040};  // 彎曲時 ADC
const int PINS[5]  = {34, 35, 36, 39, 32};             // 各手指接腳

int angle[5];  // 各手指角度（0–90°）

void setup() {
  Serial.begin(115200);  // 115200 baud，較高速率減少輸出延遲
  analogReadResolution(12);  // ESP32 設定 12-bit ADC
  Serial.println("手套控制器啟動完成");
}

void loop() {
  unsigned long t0 = millis();  // 計時開始

  for (int i = 0; i < 5; i++) {
    int raw = analogRead(PINS[i]);  // 讀取 ADC
    // map() 將 raw 從 [FLAT, BENT] 映射到 [0, 90]
    angle[i] = constrain(map(raw, FLAT[i], BENT[i], 0, 90), 0, 90);
  }

  unsigned long elapsed = millis() - t0;  // 計算延遲

  // 輸出格式：拇指/食指/中指/無名/小指 (延遲ms)
  Serial.print("角度：");
  for (int i = 0; i < 5; i++) {
    Serial.print(angle[i]);
    if (i < 4) Serial.print("/");
  }
  Serial.print("  延遲：");
  Serial.print(elapsed);
  Serial.println(" ms");

  delay(10);  // 10ms 間隔，100 Hz 採樣率
}
```

## 測試驗證

**全部伸直**：Serial 顯示 `0/0/0/0/0  延遲：X ms`，延遲應低於 **20 ms**。
**握拳**：顯示接近 `90/90/90/90/90`，允許誤差 **±8°**（個體差異）。
**單指彎曲測試**：逐根手指彎曲，確認各腳位獨立響應不互相干擾。

若延遲超過 20 ms：`analogReadResolution()` 設為 10-bit 可縮短至 **8 ms**，代價是角度解析度從 **0.088°** 降至 **0.352°**。

## 常見失敗原因

**問題**：某根手指讀值始終固定，不隨彎曲變化
**原因**：感測器脫焊或導線斷路，分壓點電壓固定
**解法**：用三用電表量感測器兩端，確認電阻隨彎曲從 8 kΩ 變化到 40 kΩ

**問題**：角度漂移，靜置也會跳動 ±10°
**原因**：ESP32 ADC 有非線性問題，特別在 0.1V 和 3.2V 附近
**解法**：讀值限制在中間 **0.5–2.8V** 範圍內，或加 10 次移動平均濾波
