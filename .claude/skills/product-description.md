# Payhip 產品頁面文案規範

## 產品頁面結構

### 標題格式
[產品名稱] — [主要功能] — [量化規格]
範例：「電容式壓力感測套件 — 兩段壓力識別 — 感測範圍 0-20 kPa」

### 第一段（產品定位）
這個產品解決什麼問題。
適合誰使用。
不超過3句。

### 第二段（內容清單）
包含什麼：
- PCB 格柏檔（可直接送廠製作）
- 完整 Arduino 程式碼（附注解）
- 材料採購清單（含型號和數量）
- 組裝說明書（含照片）
- 常見問題解答

### 第三段（規格）
列出關鍵規格（數字要具體）：
- 感測範圍：0–20 kPa
- 解析度：兩段（輕壓/重壓）
- 供電：5V（Arduino USB）
- 接腳：I2C（SDA, SCL）

### 第四段（適用範圍）
這個產品在什麼情況下有效。
什麼情況下不適用（誠實說明限制）。

## 定價邏輯
- 教學指南：$6 USD（入門門檻低）
- 完整專案包：$12 USD（有完整檔案）
- 不定期捆綁：$15 USD（兩個合購）

## 禁止的文案
- 「最好的」「唯一的」「保證成功」
- 誇大適用範圍
- 隱藏限制條件
