# 話題鉤子設計

## 什麼是有效的鉤子
讓讀者第一句話就覺得「這篇跟別的教學不一樣」

## 三種鉤子公式

### 公式1：反直覺開場
「為什麼 [常見做法] 是錯的——真正的問題是 [更深層的原因]」

範例：
「為什麼不能用 if value > 500 來觸發？
因為人的施力不是開關，是連續的意圖表達。」

### 公式2：具體失敗場景
「你的 [元件] 輸出正常，但 [預期行為] 沒發生。
問題不在感測器，在 [真正原因]。」

範例：
「你的 FSR 讀值沒問題，但裝置就是不反應。
問題不在電阻值，在你的判斷邏輯沒有考慮人施力的時間序列。」

### 公式3：設計決策點
「[情境描述]。你有兩個選擇：[選項A] 或 [選項B]。
選哪個取決於 [關鍵判斷準則]。」

範例：
「展覽裝置，觀眾靠近時改變燈光。你有兩個選擇：距離感測或壓力感測。
選哪個取決於你要捕捉的是位置還是意圖。」

## 哪些主題有高鉤子潛力
- 讀者「以為自己懂但其實不懂」的主題
- 常見錯誤有明確原因可解釋的主題
- 技術和設計思考交界的主題

## Reddit 鉤子 vs Medium 鉤子
Reddit：鉤子放在標題（一句話決定點不點）
Medium：鉤子放在第一段（讓讀者讀完整篇）

## 鉤子禁忌
- 不能是點擊誘餌（標題說的和內容不符）
- 不能是模糊問句（「你知道感測器有多重要嗎？」）
- 不能是誇大承諾（「學完這篇你就是互動設計師」）
