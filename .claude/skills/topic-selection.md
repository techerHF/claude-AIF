# 選題策略

## 選題原則

### 必須符合（全部）
- 主題範圍在：Arduino 感測器、電容式感測、觸覺回饋、PCB 設計、嵌入式系統
- 能寫出實測數據（量化結果）
- 對應到 Payhip 產品（感測器套件或完整專案）
- 不重複歷史已發主題

### 優先選擇
- 有具體量化結果的主題
- 讀者可以用現有元件複製的主題
- 解決日常生活中實際問題的主題

### 避免
- 太基礎（Hello World、LED 閃爍）
- 太抽象（概念性、沒有實作）
- 需要昂貴設備才能複製的主題

## 主題庫（輪替使用）

### A 類：感測器製作
- 電容式壓力感測陣列
- 電阻式彎曲感測器
- 矩陣式觸覺感測陣列
- 多軸力量感測器

### B 類：人機互動
- 壓力感知密碼輸入
- 手勢辨識系統
- 觸覺回饋裝置
- 防窺視輸入方式

### C 類：電路設計
- PCB 設計流程（EasyEDA）
- 類比訊號處理與放大
- 多工掃描電路
- 低功耗設計

### D 類：材料製作
- PDMS 彈性體製程
- 銅箔電極製作
- 3D 列印結構設計
- 軟性電路製作

## 選題決策流程
1. 讀取 logs/progress.json 確認已使用主題
2. 執行：bash .claude/hooks/topic-tracker.sh suggest
3. 從建議類別選擇未使用的主題
4. 執行：bash .claude/hooks/duplicate-check.sh [主題關鍵字]
5. 確認 UNIQUE 後才開始寫
6. 輸出選定主題和理由
