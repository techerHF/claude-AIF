# Whop 銷售文案寫法

## 核心原則
文案不是廣告語氣，是老師的自然延伸。
讀者應該感覺「這個人在幫我，順便告訴我還有更多」
不是「這個人在賣我東西」

## 第七層進階應用的自然帶出公式

### 公式A：問題延伸型
文章末尾，在完成基礎教學後：

「This covers the single-sensor case.
If you're building something where 2–3 sensors need to agree before triggering —
that's a state machine problem, not a sensor problem.
I wrote up the 3-state design framework here: [Whop 連結] ($9)」

### 公式B：設計深度型
「The wiring works. The threshold logic is where most projects get stuck.
I put together a design guide that covers why threshold-based triggers fail
and what to use instead — with 3 working examples.
[Whop 連結]」

### 公式C：訂閱者預告型（Medium 文末）
「If you want next month's topic (ESP32 + multi-axis haptic feedback)
before it goes public — it's in the subscriber feed.
[Whop 訂閱連結] — $5/month」

## Reddit 留言第一則固定格式
```
For anyone who wants the complete design framework with state machine examples
and the full annotated source code:

I wrote it up here → [Whop 連結] ($9)

Covers the threshold problem I mentioned, plus 2 more failure modes I've seen in real projects.
```

規則：
- 第一留言必須在發文後5分鐘內發（趁熱）
- 不超過4句話
- 不重複主文已說過的內容
- 結尾一定有具體的「$X」數字（不讓讀者猜價格）

## 訂閱者轉換文案策略

### 從免費讀者到 $9 買家
觸發點：讀者在 Reddit 問「do you have more detail on X?」
回應模板：
「Yeah, I went deeper on that in the design guide —
specifically on [X 的具體面向]. Link in my profile.」
不直接說「buy my thing」，說「went deeper on that」

### 從 $9 買家到訂閱者
觸發點：購買後的 Whop 確認信
文案：
「Thanks for grabbing this.
If you want the next topic (announced monthly) before it goes public,
the subscription feed is where I put early drafts.
[訂閱連結] — $5/month, cancel anytime.」

## 禁止在文案中出現的詞
「全面」「必備」「不買後悔」「限時」「搶購」
「只要 $X」（強調便宜感，削弱價值感）
「最好的」「唯一的」「最完整的」
